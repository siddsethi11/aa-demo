About The Demo

Scenario And Agent Roles

The demo models a customer escalation for an enterprise account that reports a billing dispute and workflow-agent sync delays. Kong stays in the middle of the UI, MCP tools, sub-agents, LLM routes, and observability path so each hop is governed and traceable.

Business Scenario

A customer escalation enters through the dashboard. The system needs to gather account context, investigate the technical issue, prepare the customer-facing follow-up, and produce an executive brief quickly.

Orchestrator

The orchestrator is the main LangGraph workflow. It discovers and calls MCP tools, creates the triage brief, invokes the support and success sub-agents, and assembles the final executive response.

Support Agent

The support agent handles the technical investigation. It uses incident and runbook tools, summarizes the current engineering posture, and recommends the immediate remediation path.

Success Agent

The success agent handles customer communication and follow-up. It uses tools to draft the customer reply, create the next task, and shape the account-team action plan.
